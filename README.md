# TOTD Patch

A compatibility patch for **The Typing of the Dead (PC, 2001)** that enables the game to run on Windows 10/11 with windowed, borderless, and fullscreen modes at arbitrary resolutions up to 4K.

The patch is a **drop-in `ddraw.dll` proxy**. No files in the game directory are modified or replaced — drop in the patch files alongside `Tod_e.exe` and remove them to uninstall.

---

## Prerequisites — dgVoodoo2

The game uses Direct3D 7, which Windows 10/11 no longer provides a hardware device for. **dgVoodoo2** re-implements D3D7 on top of D3D11 and is required for the game to run at all on modern Windows.

1. Download **dgVoodoo2** from [dege.fw.hu](http://dege.fw.hu/dgVoodoo2/dgVoodoo2/) (latest stable, e.g. 2.8.7)
2. From the archive, copy these two files into the game folder (next to `Tod_e.exe`):

   | Archive path | Rename to |
   |---|---|
   | `MS/x86/DDraw.dll` | **`dgvoodoo_ddraw.dll`** |
   | `MS/x86/D3DImm.dll` | `D3DImm.dll` (keep name) |

   > dgVoodoo2 is freeware by Dénes Soltész. It is not included here and must be downloaded separately.

---

## Installation

1. Copy **`ddraw.dll`** and **`TOTDPatch.ini`** into the game folder (next to `Tod_e.exe`).
2. Ensure `dgvoodoo_ddraw.dll` and `D3DImm.dll` are also in the game folder (see above).
3. Edit `TOTDPatch.ini` to your preference (see below).
4. Run `Tod_e.exe` normally.

**To uninstall:** delete `ddraw.dll`, `TOTDPatch.ini`, `dgvoodoo_ddraw.dll`, `D3DImm.dll`, and `dgVoodoo.conf` from the game folder.

> `dgVoodoo.conf` is auto-generated in the game folder each time the game starts, based on your `TOTDPatch.ini` settings. Do not edit it manually — changes will be overwritten.

## Configuration (`TOTDPatch.ini`)

```ini
[Display]
; Window mode: windowed | borderless | fullscreen
Mode=borderless

; Output window size (pixels). Common values:
;   1280x720   (720p)
;   1920x1080  (1080p)
;   2560x1440  (1440p)
;   3840x2160  (4K)
Width=1920
Height=1080

; KeepAspect=1 -> letterbox/pillarbox to preserve the original 4:3 ratio (recommended)
; KeepAspect=0 -> stretch to fill the window (may look distorted)
KeepAspect=1

; StretchToFit=1 overrides KeepAspect and fills the entire window regardless
StretchToFit=0

; PauseOnFocusLoss=1 -> suspend the game loop when alt-tabbed (saves CPU/GPU)
; PauseOnFocusLoss=0 -> game keeps running in the background
PauseOnFocusLoss=0

; MuteOnFocusLoss=1 -> mute game audio when alt-tabbed (independent of PauseOnFocusLoss)
; MuteOnFocusLoss=0 -> audio keeps playing in the background
MuteOnFocusLoss=0
```

### Mode

| Value | Description |
|---|---|
| `borderless` | Borderless windowed at `Width`×`Height`, centered. Alt-tab does not minimize. **(recommended)** |
| `windowed` | Standard window with title bar and border, centered. |
| `fullscreen` | Exclusive fullscreen via dgVoodoo2. |

## Game folder layout (after installation)

```
Tod_e.exe               ← original game executable (untouched)
ddraw.dll               ← this patch
TOTDPatch.ini           ← your configuration
dgvoodoo_ddraw.dll      ← dgVoodoo2 DDraw (renamed from MS/x86/DDraw.dll)
D3DImm.dll              ← dgVoodoo2 D3DImm
dgVoodoo.conf           ← auto-generated by the patch on each launch
```

## Building from source

**Requirements:** Visual Studio 2022 (any edition) with the **Desktop development with C++** workload installed.

```bat
cd patch
build.bat
```

Output: `patch\ddraw.dll`

The script sets up the x86 MSVC environment automatically and produces a 32-bit DLL to match the game.

## How it works

```
Tod_e.exe
  └─ DirectDrawCreateEx()
       └─ our ddraw.dll intercepts it
            ├─ loads dgvoodoo_ddraw.dll (falls back to system ddraw.dll)
            └─ wraps the real IDirectDraw7 in ProxyDDraw7

ProxyDDraw7::SetCooperativeLevel()
  → installs a window subclass (borderless mode) to suppress WM_ACTIVATEAPP
    minimisation and detect focus changes for pause/mute
  → applies WS_POPUP (borderless) or WS_OVERLAPPEDWINDOW (windowed) style
  → generates dgVoodoo.conf so dgVoodoo2 renders into our window

ProxyDDraw7::CreateSurface() [primary surface]
  → wraps the returned IDirectDrawSurface7 in ProxySurface7

ProxySurface7::Flip()
  → if PauseOnFocusLoss=1 and focus is lost: pumps the message queue here
    until WM_ACTIVATEAPP(true) clears the pause flag, then returns
  → delegates to dgVoodoo2's Flip for actual presentation
```

dgVoodoo2 handles all Direct3D 7 rendering, texture management, and presentation via D3D11. The patch's job is window management, focus handling, and wiring dgVoodoo2's config to the user's INI.

## Compatibility

- Tested on **Windows 11** with the US PC release (`Tod_e.exe`, dated 2001-07-23).
- DirectInput (keyboard) is unaffected — the patch does not touch input.
- dgVoodoo2 2.8.x required. Other versions may work but are untested.
